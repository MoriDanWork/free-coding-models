# Changelog v0.5.14 - 2026-06-04

### Added
- **G Theme** footer hotkey — `G` now appears in the bottom hint bar so users know they can cycle `auto → dark → light` themes from the keyboard
- **Forced background colours per theme** — light mode now paints a pure white background (`#FFFFFF`) on every line, and dark mode paints a deep dark background (`#070D18`), regardless of the terminal's native colour scheme. This means light mode looks correctly white even inside a dark-themed terminal, and vice-versa
- Initial alt-screen fill with the active theme background colour to prevent a flash of the terminal's native bg before the first TUI frame renders
- **Web dashboard detail panel slide-in animation** — `panelOpen` CSS class triggers `translateX(0)` when a model is selected in the detail panel

### Changed
- Theme background fill uses `\x1b[48;2;R;G;Bm` + `\x1b[K` on every line end and `\x1b[J` on the last line so the erase-to-EOL and erase-below operations respect the theme's background colour
