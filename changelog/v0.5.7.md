# Changelog v0.5.7 - 2026-06-01

### Added
- Added Web Dashboard Smart Recommend parity with the TUI scoring engine: a three-question wizard, 10-second analysis phase, Top 3 recommendations, and Pin + endpoint-install actions.
- Added the Web endpoint target picker in the header, persisted through the shared `preferredToolMode` config setting and shareable via `?toolMode=...` URLs.
- Added per-row and Detail Panel endpoint install actions so users can write the selected provider/model endpoint into supported tool configs directly from the Web Dashboard.
- Added `/api/install-endpoint` for Web endpoint installation, backed by the shared `installProviderEndpoints` core engine.
- Added `/api/recommend` and `/api/telemetry/event` Web routes for Smart Recommend and safe Web telemetry mirroring.
- Added an incompatible-target fallback modal that lets users switch endpoint target or choose a similar compatible model.

### Changed
- Changed Web Dashboard tool actions to configure endpoints only. The Web UI no longer starts external tools or installs missing tool binaries; users launch their tools themselves after configuration.
- Kept `/api/launch` as a backward-compatible alias for endpoint installation, but it no longer spawns any external process.
- Limited Web endpoint target cycling to tools supported by the endpoint installer, avoiding launch-only modes in the Web picker.
- Updated README and the M3 parity plan to document the endpoint-install behavior.

### Fixed
- Fixed the Web M3 hook ordering issue by stabilizing toast callbacks before hooks consume them.
- Fixed the Xcode launcher instruction branch to use `launchPlan.baseUrl` / `launchPlan.apiKey` instead of out-of-scope variables.
- Fixed endpoint install target metadata so OpenCode Web canonicalizes to the shared OpenCode config path.
